Integration using TCPIP nodes

Learn how to use TCP/IP nodes for integration with external applications using TCPIP sockets.
OVERVIEW

IBM Integration Bus provides a way to integrate with external application through the TCP/IP sockets. You can use TCPIPServer nodes that use server connection type and TCPIPClient nodes utilising client connection type. The connection can be specified using port and host (reqyired by Client nodes) or using TCPIP configurable services names. This tutorial uses configurable services as a convenient way to manage the TCP/IP connections.

This tutorial includes an application 'TCPIPServer' that simulates a TCP/IP Server that is configured to send the data over the TCPIP sockets. The main flow in the server application is driven by an XML message arriving on the HTTP input node. Once the flow with TCP/IP Server node is deployed the TCPIP server is started automatically.

It also demonstrates how a flow can be driven by the TCPIP Client input node. By default this node does not open a client connection. The way to configure it properly is to use a TCPClient configurable service to enable at least one client connection to be opened when the flow is deployed. The same configurable service also defines the server host name and port to connect to.

An important part of TCPIP input nodes configuration is setting up the proper record detection algorithm. This tutorial demonstrates usage of 'Parsed Record Sequence' record detection using XML messages.

You can also find examples of usage for Trace, FileOutput, Mapping, HTTPInput and HTTPReply nodes.

This tutorial takes approximately 10 minutes.
