Click Import to see appllications TCPIPServer, TCPIPClient and shared library DataLibrary in your workspace.

A message flow TCPIPProvider.msgflow in the application TCPIPServer receives the XML message through HTTP Input node. The message is parsed and passed to the TCPIP Server Output node to send the data to the client connected on port 7778. The TCPIP Server Output node is configured using configurable service TCPIPTutorialServerCF. To avoid deployment errors the configurable service must be created on the target deployment integration node before the application is deployed. Trace nodes are used in this tutorial to trace the data flow into the common file while the data travels across the message flows.

Application TCPIPClient contains a message flow TCPIPReceiver.msgflow which uses the TCPIP Client Input Node to receive the data from the server. The TCPIP Client Input Node is configured with the configigurable service TCPIPTutorialClientCF that specifies the host, port, minimum number of automaticaly opened client connections. This message flow also contains a mapping node using nested maps to that transforms the message with arrays to a message with a different format. Finally the message is written to a file through the File Output node.

Shared library DataLibrary contains a message models for the input and output message formats SaleReport.xsd, SaleMessage.xsd as well as test message that will be used to initiate the tutorial flow TestSaleMessage.xml. It also contains a batch file that contains commands that need to be executed in the IBM Integration Console to create both required configurable services configure_tcpip.bat.

Exploring the flows and creating configurable services

Open the message flow TCPIPProvider.msgflow in the application TCPIPServer by double-clicking the flow.The flow sticky notes attached to the nodes provide details of node configuration. You can explore the node configuration in the Properties view by selecting nodes on the editor canvas. Note that the 'Connection details' property of TCPIP Server Output Node points to a configurable service TCPIPTutorialServerCF.

If you open the message flow TCPIPReceiver.msgflow in the application TCPIPClient by double-clicking the flow you can also view more information about particular node configuration using the sticky notes. Again note that the 'Connection details' property of TCPIP Client Input Node points to a configurable service TCPIPTutorialClientCF. Also see how the 'Records and Elements' tab allows you to configure the record detection.

It is important to remember that to successfully deploy your application to the integration server you need to create two required configurable services. To create the configurable service on the target integration node TESTNODE_user_id used by the flow excercizer run the provided configure_tcpip.bat file in the IBM Integration Console.

Tips:You may need to adjust the ports in the configurable services commands for your system if the 7778 can not be used. You can also change the file paths specified by the Trace and FileOutput nodes.

Next, you will deploy and run the tutorial to see how the data travels from the TCPIPServer to the TCPIPClient and then to the file on the file system. Click Run to see those steps.

Follow these steps to complete the tutorial

Deploy and run the tutorial flows:

  1. Open the flow TCPIPProvider.msgflow from the application TCPIPServerProviderby double clicking on it. Click the Flow Exerciser icon to start recording the message path through the flow.
  2. Click the Send Message icon to select a message to send to the flow.
  3. Add a new input message to the tree on the left and check the Import from file option. Then select the file from the workspace /DataLibrary/TestSaleMessage.xml. Then click Send button to invoke the flow.
  4. After the request message is processed, the message path is automatically highlighted on the message flow.

Verify the flow execution:

  1. Click on any connection to see the data that passed through that connection.
  2. You can also open a trace file C:\temp\TCPIP_TRACE.txt specified by the Trace nodes to see how the message travels through the message flows.
  3. In the same output directory you should see the output file C:\temp\TCPClientTutorialOutput.txt created by the file output node. The file should contain the transformed message SalesReport.
  4. To stop recording the message path through the flow, click the Return flow to edit mode icon .
  5. You can click Clean up action to remove the tutorial source code from the workspace. You need to manually go to the target integration servers and delete the deployed applictaions using Delete context menu.
